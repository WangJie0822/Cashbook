## 踩坑沉淀（黑盒驱动经验）

1. **收入分类有两个"报销"**：第一行"报销"`[142,559]` = 系统报销款（-2002，protected，needRelated，选中后显示"关联的记录"入口）；第二行"报销"`[142,730]` = 用户自建同名一级分类（非 needRelated，无关联入口）。选错（点第二行）会导致关联入口消失。**判别：选中后是否出现"关联的记录"chip**。退款款 -2001 同理（系统"退款"在第一行）。
2. **截图全白**：`android screen capture` 对该 Compose app 输出全白（Surface 截图限制），全程靠 `android layout` JSON dump 驱动（更可靠，含 text/center/state）。
3. **记账金额计算器键盘态遮挡上半界面**：进记账界面默认弹计算器，遮住 tab/分类网格；需先输金额 + 点计算器"确认"收起，才见 tab（支出/收入/转账）与分类网格。
4. **资产编辑确认 FAB 被软键盘遮挡**：余额输入后软键盘占底部、遮住确认 FAB `[964,2158]`，需先 `keyevent 4`(BACK) 收键盘再点确认（一次 BACK 在键盘显示时只收键盘不返回页面）。
5. **DoKit 浮窗 pm clear 后重置左上 `[76,139]`** 拦截"菜单"`[74,147]`（BUG-2），需 `input swipe` 拖到底部 `[76,1681]` 后菜单方可点。
6. **分类网格 BUG-1 修复后有 text**：上轮报告 §3 称"分类网格图标无 text/contentDesc"，本轮实测 BUG-1 修复后分类格子带 text 标签（餐饮/报销等），命中容易。
